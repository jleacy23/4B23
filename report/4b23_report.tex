\section{Methodology}
\subsection{Routing}
The first step was to determine the proportion of capacity that should be allocated to each route joining two nodes in the network. The number of channels available to each link is approximately 
\begin{equation}
n_i = n_{busiest} \times \frac{C_i}{C_{busiest}}
\label{eq:num_channels}
\end{equation}
where \(n\) is the number of channels and \(C\) is the capacity of a link. Hence an optimal routing maximised the total channels (and thus total capacity) by minimising the usage of the busiest link. This was framed as the multi-commodity flow problem and solved with linear programming given the demands between nodes and the maximum capacities of each link. The result was the relative usage of every route through the network.
\subsection{Channel Assignment}
Equation \ref{eq:num_channels} indicates that the maximum number of channels should be assigned to the busiest link. Here the EDFA limits this to 25. Channels were then assigned to each route passing through the busiest link in proportion to their usages. One of these routes was then used as a reference to assign channels to those not passing through the busiest link as
\begin{equation}
    n_i = n_{ref} \times \frac{C_i}{C_{ref}}.
\end{equation}
A final check was then performed to ensure every link had at most 25 channels.
\subsection{Calculating NSR of a Link}
The NSR due to amplifier noise over all spans is given by
\begin{equation}
    NSR_{ASE} = \frac{N_s R_s}{P_{TX}}10^{G/10}h\nu(10^{G/10}-1)
    \label{eq:nsr_amp}
\end{equation}
where \(NF\) is the noise figure of the EDFA, \(N_s\) is the number of spans, \(G\) is the amplifier gain in dB, \(R_s\) is the symbol rate, and \(P_{TX}\) is the transmitted power per channel in mW. This assumed a brick-wall shaped pulse. The NSR due to non-linear effects is
\begin{equation}
    NSR_{NLI} = N_s\frac{N_{NLI}}{G_{TX}} = N_s C_{NLI}G_{TX}^2 = \frac{N_s C_{NLI} P_{TX}^2}{R_s^2}
    \label{eq:nsr_nli}
\end{equation}
where \(C_{NLI}\) is the non-linear coefficient of the length of fiber in a span. These can then be summed to find the total NSR of a link.
The maximum span length \(L_{s,\max}\) can be found from the unsaturated gain \(G_0\) and fiber attenuation \(\alpha\) as \(L_{s,\max} = \frac{G_0}{\alpha}\). To ensure amplifiers can be placed at the same point along each direction of the link, an integer number of spans should be enforced. Thus for a link length \(L_T\), \(N_s = \lceil \frac{L_T}{L_{s,\max}} \rceil \) and the span length \(L_s = \frac{L_T}{N_s}\). This will result in each span having a net gain greater than 0 dB which, meaning the transmitted power changes between spans. This is difficult to optimize for so the variable attenuator can be used at the input to each amplifier or the launch power adjusted to saturate the amplifier so there is no net gain per span. The first approach requires a sweep over a range of \(P_{TX}\). The true gain of each amplifier (accounting for saturation) was calculated as
\begin{equation}
    G = G_0 - 10\log_{10}(1 + \frac{n_i P_{TX}}{P_{SAT}})
\end{equation}
where \(P_{SAT}\) is the saturation power in mW. No net gain across spans meant that this gain was the same for all amplifiers in the link. Equation \ref{eq:nsr_amp} was then used to find the amplifier NSR and combined with the non-linear noise from Equation \ref{eq:nsr_nli}. Note that \(C_{NLI}\) is fixed by the span length and fiber type. The \(P_{TX}\) with the best NSR was then chosen. The second approach fixes the gain to be \(\alpha L_s\) then finds the required \(P_{TX}\) to achieve it and its corresponding NSR. The approach with the best NSR was then chosen.
\subsection{Calculating NSR of a Route}
The total NSR present along a route is calculated by summing the NSRs of each link \(NSR_i\) it passed through. As \(P_{TX}\) was calculated for each link individually, the \(P_{TX}\) of one link may be lower than that of the following link along the route. In this case, an additional amplifier is needed at the start of the next link to normalize to its optimal \(P_{TX}\). This incurs extra NSR from the amplifier which must be added to the route total. When the route terminates at a transceiver, \(P_{TX}\) of the last link must be checked against the minimum operating power for the transceiver. If it is too low then an extra amplifier must be added, further increasing the NSR. Finally the transceiver NSR is included.
\subsection{Validating a Route}
A route in the network was declared valid for a given transceiver mode if it could deliver an output power greater than the minimum of the transceiver and an NSR lower than the maximum of the transceiver. Fiber type was kept the same throughout the network but the transceiver mode could vary between routes as they operated on individual channels. For each fiber type, the NSR and output power of every route was calculated then checked against the transceiver constraints. The transceiver mode that had the highest data rate and was valid for the route \(D_{\max}\) was then used to find the capacity of a route \(C_r = n_r D_{\max}\), where \(n_r\) is the number of channels assigned to the route.
\subsection{Optimization and Measuring Resilience}
The total capacity of a network was found by summing the capacities of every route. This was repeated over all fiber types to find the highest capacity and thus the optimal design for each network. If the highest capacity was achieved by multiple fiber types then that which minimized the number of amplifiers was chosen.
